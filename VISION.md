# VISION.md

This file defines the durable purpose and constraints for the OpenVibeAI-data repository.

## Purpose

OpenVibeAI-data is the **GitHub Pages data repository** for the OpenVibeAI product suite. It hosts static JSON data that the OpenVibeAI website and other consumers fetch at runtime. By separating data from the website code, content updates can be published immediately without website redeployments.

## Products Served

| Product | Data Path | Content |
|---|---|---|
| Blog & News | `data/blog_and_news/` | AI-curated news digests and editorial blog posts |
| Voice Typer | `voice-typer/` | Voice Typer download metadata |

## Blog & News Structure

The canonical data layout follows `BLOG_SCHEMA.md`:

```
data/blog_and_news/
  index.jsonl          # compact discovery feed (type, slug, title, excerpt, publishedAt, path)
  blog/                # full blog documents (editorial / manually authored)
    {slug}.json
  news/                # full news documents (auto-generated by RSS publisher workflow)
    {slug}.json
```

### Schema Rules

- Each `index.jsonl` line is a JSON object with: `type` (`blog` | `news`), `slug`, `title`, `excerpt`, `publishedAt`, `path`.
- Individual news documents follow the `AiNewsStory` shape described in `AI_NEWS_SCHEMA.md`, with `type: "news"` and `websiteUrl` pointing to `/blog-and-news/news/{slug}`.
- Individual blog documents follow `BLOG_SCHEMA.md`.
- The `index.jsonl` is sorted newest-first by `publishedAt`.

## Non-Goals

- No backend, database, CMS, or authentication.
- No server-side rendering or build-time import of data into the website.
- No dynamic APIs — all data is static JSON over HTTPS via GitHub Pages.

## Constraints

- All file writes must be idempotent and atomic.
- Blog entries (manually authored) are never deleted or overwritten by automated workflows.
- Non-workflow news entries already in `index.jsonl` are preserved on re-export.
- The RSS publisher workflow (`ai-news-rss-top-impact-publisher`) owns the `news/` directory and news entries in `index.jsonl`.
